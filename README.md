# 🍽️ Karmic Canteen – Smart Meal Management System

Karmic Canteen is a smart meal management and demand forecasting platform designed to optimize canteen operations, reduce food waste, and support surplus food distribution. 

The system consists of a web dashboard for administrators and employees, a Flutter-based mobile application, and a local Python machine learning server that forecasts meal demand using historical consumption patterns.

---

## 📋 Problem Statement

Corporate and institutional canteens face significant challenges in balancing meal preparation with actual daily demand. A lack of visibility into daily employee attendance, working modes (Office vs. Work from Home), and specific meal selections leads to:
* **Substantial Food Waste:** Over-preparing meals due to unpredictable daily attendance.
* **Operational Inefficiencies:** Canteen staff struggling to manage menus, deadlines, and portion sizes manually.
* **Missed Social Impact:** Lack of a streamlined communication channel to notify local non-governmental organizations (NGOs) when surplus food is available, causing edible food to go to waste instead of being distributed.

Karmic Canteen solves these issues by providing employees with easy pre-selection tools, giving administrators real-time tracking dashboards, utilizing machine learning to forecast next-day meal counts, and offering a direct notification system to dispatch surplus food alerts to NGOs.

## ✨ Current Features

### 👤 Employee Portal (React Web & Flutter Mobile)
* **Weekly Meal Selection:** Employees can choose their breakfast, lunch, and snacks on a calendar view for the upcoming week.
* **Working Mode Selector:** Switch between "Office" and "Work From Home" mode, automatically syncing selections.
* **Real-time Deadlines:** A visual countdown timer that prevents selections or changes after the daily cutoff time.
* **Multilingual UI:** Switch languages dynamically between English, Hindi, and Kannada.

### 👨‍💼 Admin Dashboard (React Web & Flutter Mobile)
* **Live Analytics:** Track total registered employees, today's active participants, total meals ordered, and a percentage breakdown of orders.
* **Menu Management:** Create, update, and publish menus for breakfast, lunch, and snacks.
* **Interactive AI Predictions (Web Only):** Request next-day predictions, weekly forecasts (N-day), and trend insights generated by the local Python ML server.
* **Reports Generator:** Generate and export detailed meal selection and attendance reports to CSV and PDF formats (using jsPDF).
* **Surplus Food Notification (Flutter Mobile Only):** Input leftover/surplus meal counts to send automated email alerts directly to registered NGOs.
* **Announcements:** Broadcast festival and special event announcements to all employees.

### 🤖 Machine Learning demand Forecasting (Python Flask API)
* **Historical Data Syncing:** Automatically fetches Firestore meal selections and updates a local cached dataset (`canteen_history.csv`).
* **LightGBM Demand Models:** Trains individual forecasting models for each menu item to project tomorrow's popular choices.
* **Forecast Ranges:** Generates predicted meal counts, opt-in rates, lower/upper bounds, and confidence intervals.
* **Trend Insights Engine:** Analyzes day-of-week averages, holiday consumption deviations, and top-performing menu items.

## 🚀 Getting Started

### Prerequisites
* **Node.js** v18+ and **npm**
* **Flutter SDK** v3.0.0+ and Dart
* **Python** v3.8+ and **pip**
* A configured **Firebase Project** (Firestore, Authentication, and Cloud Functions enabled)

---

### 1. React Web Frontend Setup
1. Navigate to the web frontend directory:
   ```bash
   cd madhwa-hackathon/frontend
   ```
2. Install the node packages:
   ```bash
   npm install
   ```
3. Start the Vite development server:
   ```bash
   npm run dev
   ```
4. Access the web interface at `http://localhost:5173`.

---

### 2. Python ML Backend Setup
1. Navigate to the machine learning directory:
   ```bash
   cd madhwa-hackathon/canteen_management_system/canteen_management_system
   ```
2. Install Python dependencies:
   ```bash
   pip install -r requirements.txt
   ```
3. Configure your Firebase credentials (place your Service Account JSON file in the folder and update `config.py` / environment variables).
4. Run the Flask API server:
   ```bash
   python app.py
   ```
   *(Alternatively, on Windows, double-click `START.bat` or `START_ML_SERVER.bat`)*
5. The ML API server will run locally at `http://localhost:5000`.

---

### 3. Flutter Mobile Frontend Setup
1. Navigate to the mobile application folder:
   ```bash
   cd flutter-frontend/karmic_canteen_app
   ```
2. Fetch Dart dependencies:
   ```bash
   flutter pub get
   ```
3. Run the application on your connected emulator or device:
   ```bash
   flutter run
   ```

## 🚀 Future Roadmap

The following features are planned for future releases of Karmic Canteen:
* **Automated RFID Meal Collection:** Integrate physical RFID readers (such as an ESP32 microcontroller and MFRC522 scanner) to automatically mark meals as collected in the Firestore database, preventing duplicate collections and human tracking errors.
* **NGO Portal & Distribution Portal:** Build a dedicated web and mobile login dashboard for NGO users. This will enable them to manage their organizational profiles, view live surplus notifications, claim/reserve surplus meals, schedule pickup times, and download automated social impact reports.
* **Cloud-Hosted AI Pipeline:** Migrate the local Python Flask backend to a hosted cloud environment (e.g., Google Cloud Run / Firebase Functions) to automatically trigger demand predictions and update Firestore daily, eliminating the requirement to run a local backend.
* **Automated Food Waste Prediction:** Develop predictive ML algorithms that specifically calculate expected food waste and leftovers by correlating meal selection statistics with historical attendance patterns and holiday impacts.
* **Automated Notifications & Reminders:** Set up active email/SMS/push notifications (via Firebase Cloud Messaging) to remind employees to make selections before deadlines.
